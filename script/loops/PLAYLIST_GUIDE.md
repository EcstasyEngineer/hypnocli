# Loop Script Playlist Guide

This document describes the ending states of each loop script and how they prime for subsequent themes. Used for:
1. **User-facing descriptions** - what state to expect
2. **Discord bot playlist generation** - automated sequencing

---

## State Categories

Scripts leave listeners in one or more of these state categories:

| State | Description |
|-------|-------------|
| **receptive** | Open, suggestible, accepting |
| **empty** | Blank, no thoughts, void |
| **obedient** | Compliant, following commands |
| **owned** | Belonging to authority, possessed |
| **desperate** | Aching, needing, craving |
| **identity-shifted** | Transformed sense of self |
| **aroused** | Sexually activated, edged |

## Bot Implementation Notes

For Discord bot playlist generation:

1. **State tracking**: Track cumulative ending states across playlist
2. **Transition scoring**: Use matrix to score proposed next tracks
3. **Intensity ramping**: Don't jump from Light to Extreme
4. **User preferences**: Weight toward user's selected themes
5. **Avoid repeats**: Same script shouldn't appear twice in short playlist

### Suggested Algorithm
```python
def next_track(current_state, user_themes, played_already):
    candidates = [t for t in all_tracks if t not in played_already]
    candidates = [t for t in candidates if transition_score(current_state, t) >= YELLOW]
    candidates = sorted(candidates, key=lambda t: (
        theme_match_score(t, user_themes),
        transition_score(current_state, t),
        intensity_appropriateness(current_state, t)
    ), reverse=True)
    return candidates[0]
```
